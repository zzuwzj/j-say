# J-Say 语料库后台管理系统 — 需求文档

## 1. 项目概述

### 1.1 背景

J-Say 是一款 IELTS 口语练习 Web 应用，语料库（话题、词汇、示例回答等）是核心内容资产。当前语料以静态 JSON 文件硬编码在前端，存在以下问题：

- 新增/修改话题需要修改代码并重新部署
- 无法在线管理语料内容
- 无法统计语料使用情况和效果
- 缺少版本管理，无法回退

### 1.2 目标

构建语料库后台管理系统（Corpus Admin），实现语料的在线管理、发布与同步，让用户直接维护语料内容，无需修改代码。

### 1.3 目标用户

个人用户，自行管理语料库内容，无审核流程。

## 2. 功能需求

### 2.1 功能架构

```
语料库后台管理系统
├── 语料管理
│   ├── 话题管理（CRUD、批量操作、关联管理）
│   ├── 词汇管理（CRUD、批量操作、话题关联）
│   ├── 分类标签管理
│   └── 语料导入导出
├── 数据统计
│   ├── 语料总览（数量、分布、趋势）
│   ├── 使用分析（练习频次、热门话题）
│   └── 质量分析（评分关联、完成率）
├── 系统管理
│   ├── 操作日志
│   └── 系统配置
└── 内容发布
    ├── 版本管理
    ├── 发布管理
    └── 客户端同步
```

### 2.2 P0 — 核心功能

#### 2.2.1 话题管理

**列表页**

- 分页展示所有话题，支持以下筛选条件：
  - Part 筛选（Part 1 / Part 2 / Part 3）
  - 分类筛选（人物/经历/地点/物品/事件/社会/日常生活/教育/科技/文化）
  - 难度筛选（简单/中等/困难）
  - 状态筛选（草稿/已发布/已下架）
  - 关键词搜索（标题、内容）
- 列表字段：ID、Part、分类、标题、难度、状态、更新时间、操作
- 支持排序：按更新时间、创建时间排序
- 批量操作：批量删除、批量修改分类/难度/状态

**创建/编辑话题**

- Part 选择（1/2/3）
- 分类选择
- 难度选择
- 标题（必填，限 200 字符）
- 内容描述（必填，富文本）
- 问题列表：
  - 每个问题支持编辑文本
  - 每个问题可添加示例回答（简单版 + Band 7 版）
  - 支持拖拽排序
  - 支持增删问题
- Part 2 专属字段：
  - 提示点列表（prompts）
  - 独白示例（简单版 + Band 7 版）
- Part 3 专属字段：
  - 关联 Part 2 话题（下拉选择）
- 保存为草稿 / 直接发布

**话题详情页**

- 展示话题完整信息
- 显示关联词汇
- 显示使用统计（被练习次数、平均评分）
- 状态切换操作

#### 2.2.2 词汇管理

**列表页**

- 分页展示，支持筛选：
  - 分类筛选
  - 关键词搜索（单词、释义）
- 列表字段：ID、单词、释义、分类、关联话题数、操作
- 批量操作：批量删除、批量修改分类

**创建/编辑词汇**

- 单词/短语（必填）
- 释义（必填）
- 例句
- 分类选择
- 关联话题（多选，从话题库中选择）
- 保存为草稿 / 直接发布

#### 2.2.3 分类标签管理

- 分类列表（CRUD）
- 每个分类包含：标识符（slug）、中文名称、英文名称、排序权重、启用状态
- 系统预置分类不可删除，可禁用
- 支持自定义分类的增删改

#### 2.2.4 语料导入导出

**导入**

- 支持 JSON 文件导入（与前端数据格式兼容）
- 导入前预览：展示待导入数据条数，标记重复项
- 重复处理策略：跳过 / 覆盖 / 保留两者
- 导入结果报告：成功数、失败数、跳过数

**导出**

- 支持按筛选条件导出（全部 / 按 Part / 按分类 / 按状态）
- 导出格式：JSON（与前端 `src/data/topics/` 和 `src/data/vocabulary.ts` 格式兼容）
- 导出内容可选：话题、词汇、或两者

### 2.3 P1 — 增强功能

#### 2.3.1 数据统计

**语料总览**

- 话题总数（按 Part、分类、难度、状态分布）
- 词汇总数（按分类分布）
- 新增趋势图（日/周/月）
- 语料完成度：有示例回答的问题占比

**使用分析**

- 热门话题 TOP 20（按练习次数）
- 冷门话题（近 30 天零使用）
- 词汇被引用次数排行
- 各分类练习频次分布

**质量分析**

- 话题评分关联：各话题的平均自评分
- 高分话题特征分析
- 低分话题预警

#### 2.3.2 版本管理与发布

**版本管理**

- 每次语料变更自动生成版本记录
- 版本信息：版本号、变更摘要、变更人、时间
- 支持版本对比（diff）
- 支持回滚到指定版本

**发布管理**

- 发布语料包：将已发布的语料打包为一个发布版本
- 发布状态：未发布 / 已发布 / 已回滚
- 发布记录列表

**客户端同步**

- 前端应用启动时检查语料版本
- 增量同步：仅下载变更的语料
- 本地缓存版本号，避免重复下载

#### 2.3.3 系统管理

**操作日志**

- 记录所有关键操作：语料变更、发布操作
- 日志字段：操作类型、操作对象、操作时间、操作详情
- 支持按操作类型、时间范围筛选

**系统配置**

- 语料版本号管理
- 导入限制（单次最大条数）

### 2.4 P2 — 优化功能

#### 2.4.1 语料质量辅助

- 话题完整性检查：必填字段校验、示例回答覆盖率
- 重复检测：相似话题/词汇提醒
- AI 辅助生成：根据话题标题自动生成示例回答（可选）

#### 2.4.2 批量高级操作

- 批量修改话题难度
- 批量关联词汇到话题
- 批量调整排序

## 3. 语料状态流转

```
              ┌──────────┐
    创建      │   草稿    │
   ────────→  │  draft   │
   │          └────┬─────┘
   │               │ 发布
   │               ▼
   │          ┌──────────┐
   │          │  已发布   │
   │          │published │
   │          └────┬─────┘
   │               │
   │       ┌───────┴───────┐
   │       │ 下架           │ 重新编辑
   │       ▼               │
   │  ┌──────────┐         │
   │  │  已下架   │         │
   │  │ offline  │─────────┘
   │  └──────────┘
   │
   └──────── 重新编辑（回到草稿）
```

简化流转：草稿 → 发布 → 下架，无审核环节。用户直接操作。

## 4. 数据模型

### 4.1 话题（Topic）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 自动 | 主键 |
| part | 1 \| 2 \| 3 | 是 | IELTS Part |
| category | string | 是 | 分类标识 |
| title | string | 是 | 话题标题，限 200 字符 |
| content | string | 是 | 话题描述 |
| prompts | string[] | 否 | Part 2 提示点 |
| questions | string[] | 是 | 问题列表 |
| examples | QuestionExample[] | 否 | 每题示例回答 |
| part2Example | QuestionExample | 否 | Part 2 独白示例 |
| relatedPart2Id | number \| null | 否 | Part 3 关联的 Part 2 话题 ID |
| difficulty | 'easy' \| 'medium' \| 'hard' | 是 | 难度 |
| isCustom | boolean | 是 | 是否用户自定义 |
| isFavorite | boolean | 是 | 是否收藏 |
| status | 'draft' \| 'published' \| 'offline' | 是 | 状态 |
| createdAt | number | 自动 | 创建时间戳 |
| updatedAt | number | 自动 | 更新时间戳 |
| publishedAt | number \| null | 否 | 发布时间戳 |
| version | number | 自动 | 版本号，每次修改 +1 |

### 4.2 词汇（Vocabulary）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 自动 | 主键 |
| word | string | 是 | 单词/短语 |
| definition | string | 是 | 释义 |
| example | string | 否 | 例句 |
| category | string | 是 | 分类标识 |
| relatedTopicIds | number[] | 否 | 关联话题 ID 列表 |
| isCustom | boolean | 是 | 是否用户自定义 |
| isFavorite | boolean | 是 | 是否收藏 |
| status | 'draft' \| 'published' \| 'offline' | 是 | 状态 |
| createdAt | number | 自动 | 创建时间戳 |
| updatedAt | number | 自动 | 更新时间戳 |
| version | number | 自动 | 版本号 |

### 4.3 分类（Category）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 自动 | 主键 |
| slug | string | 是 | 标识符，如 'person' |
| nameZh | string | 是 | 中文名称 |
| nameEn | string | 是 | 英文名称 |
| sortOrder | number | 是 | 排序权重 |
| enabled | boolean | 是 | 是否启用 |
| isSystem | boolean | 是 | 是否系统预置 |
| createdAt | number | 自动 | 创建时间戳 |

### 4.4 操作日志（AuditLog）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 自动 | 主键 |
| action | string | 是 | 操作类型（create/update/delete/publish/import/export） |
| targetType | string | 否 | 操作对象类型（topic/vocabulary/category） |
| targetId | number | 否 | 操作对象 ID |
| detail | string | 否 | 操作详情（JSON） |
| createdAt | number | 自动 | 操作时间戳 |

### 4.5 语料版本（CorpusVersion）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 自动 | 主键 |
| version | string | 是 | 版本号，如 '1.2.0' |
| summary | string | 否 | 变更摘要 |
| topicCount | number | 是 | 话题数量 |
| vocabCount | number | 是 | 词汇数量 |
| status | 'unpublished' \| 'published' \| 'rolled_back' | 是 | 发布状态 |
| publishedAt | number \| null | 否 | 发布时间戳 |
| createdAt | number | 自动 | 创建时间戳 |

## 5. 非功能需求

### 5.1 性能

- 话题列表页加载时间 < 1s（1000 条数据）
- 单次导入支持最大 5000 条语料
- 导出操作响应时间 < 3s

### 5.2 可用性

- 响应式布局，支持 1280px+ 宽度
- 列表页筛选条件支持 URL 参数持久化
- 批量操作前二次确认

### 5.3 数据安全

- 话题和词汇支持软删除（30 天内可恢复）
- 发布操作支持回滚

## 7. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 前端同步兼容性 | 旧版客户端无法解析新语料格式 | 版本号控制，增量同步，保留向后兼容 |
| 导入数据质量 | 外部数据格式不规范 | 预览阶段严格校验，提供错误报告 |
| 语料数据量增长 | 列表查询变慢 | 分页查询 + 索引优化 |
